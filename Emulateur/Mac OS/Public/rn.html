OS X Mercury Release Notes

SE51 OS X Mercury
Release notes


Introduction

On Friday, May 18 2007, SE51 has launched the final code for the long anticipated personal homepage project named OS X Mercury, formerly code-named "Asteroid". The Mercury website closesly resembles appearences and functionalities that come from Mac-maker Apple's operating system Mac OS X. The most notable feature in its premiering release is Time Machine - a 3D environment that allows the visitor to literally browse through older websites of SE51 and then open them in the built-in browser of the environment.

The first version has been in the making for more than a year and consists of about 30,000 lines of code. Although based on platform independent technologies such as JavaScript and DHTML, Mercury has been developed entirely for Mozilla's Firefox browser. SE51 points out that Firefox is the most advanced platform independent browser available, therefore Firefox has been picked as the browser for which Mercury has been designed for. Most of the technologies presented, such as the Oxygen framework, is entirely based on Open Source technology, and is credited in the about box.

OS X Mercury has recently been Beta tested by a small number of people, and has now been deemed stable for release. SE51 is now looking forward to the next version, and continuous bugfixing in the current release. Which features are planned for the next release have not been disclosed, however.

The completion of the Mercury project completes the missing digital content that was planned to be found on SE51.net. With an ever growing content of digital content such as music and photos, Mercury provides an innovative approach of displaying this content online.

Known issues

  • The dock may disappear when applications are opened.
  • I have been unable to repro this issue on the machines I have tested Mercury on, therefore I have not yet identified the root cause for this issue, but I am working on a fix for this.
    When a window is dragged over the area of the dock, the dock "holder" goes into the back, but the icons stay in front, and float above the window. The icon still zooms, but the text above it is also in the back like the dock "holder" image.
  • Icons in Finder and Dock are not smooth on Windows machines. This is due to a lack of smoothing support on all Windows machines. It is uncertain if this can be fixed anytime soon. However, first versions of Firefox 3 show an improved smoothing support when it comes to scaling. If you are a bleeding edge tester, and you need to see smooth icons on Windows machines, you should try Mercury in the Alpha versions of Firefox 3. Also, SVG support for Mercury is pursued. Once implemented, all icons can be independently scaled. Icon smoothing is built in to Mac OS X. On those machines the dock will appear smooth from the beginning.
  • The dock is slightly out of place directly after opening Mercury. This appears to be a bug in the rendering function of the dock script. Help on this would be appreciated. Fix for now: move the mouse over the dock once, then leave. This will bring the dock into the correct alignment.

 

Acquiring the source code

SE51 is developing a Mercury exclusive 'application' named "MCode", which will be introduced to the environment at a later stage. This application allows potential visitors to look into the code that is running the environment. This includes the framework and all applications, as well as 'system services' such as the Finder and Dock.
